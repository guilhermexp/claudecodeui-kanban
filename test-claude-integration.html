Claude Code UI - Integration Test

✅ Features Implemented:

  • ✅ Dropdown selector for CLI provider (Codex AI / Claude Code)
  • ✅ State management for CLI provider selection
  • ✅ Persistence of user preference in localStorage
  • ✅ Dynamic UI labels based on selected provider
  • ✅ Claude normalizer for message processing
  • ✅ Backend WebSocket handlers for Claude events
  • ✅ Separate session management for each CLI

How to Test:

  1. Open http://localhost:5892
  2. Login with credentials
  3. Click on the dropdown next to the assistant button
  4. Select "Claude Code" from the dropdown
  5. Click "Start Claude Code Session"
  6. Send a message to test Claude integration
  7. Switch back to "Codex AI" to test the other integration

Expected Behavior:

  • The dropdown shows both "Codex AI" and "Claude Code" options
  • The button text changes to reflect the selected provider
  • The header shows "CLAUDE" when Claude is selected, "CODEX" when Codex is selected
  • Sessions are independent for each provider
  • User preference is saved and restored on page reload
